1.异常
    运行期检测到的错误被称为异常。
2.异常处理
    如何处理程序运行过程中发生的错误(异常情况)
    注意：处理并不是规避错误的出现，只是在出现错误的时候，我们可以提供解决方案，让程序继续执行下去
    最基本语法：
        try:
            可能出现异常的代码块
        except:
            如果出现异常，对异常进行处理的代码块
    执行过程
        如果没有发生异常，程序会忽略except中的代码，继续向下执行
        如果发生异常，发生异常后面的代码会被忽略，直接进入到except，执行except中的代码
        程序异常处理后的其它代码会正常执行

3.捕获不同类型的异常
    如果在对异常进行处理的时候，不同类型的异常，向给与不同的处理方式
    那么可以通过except将不同类型的异常进行捕获
    语法：
        try:
            可能出现异常的代码块
        except 异常类型1 as 变量1:
            针对类型1异常的处理方式
        except 异常类型2 as 变量2:
            针对类型2异常的处理方式
        except (异常类型3，异常类型4，异常类型5) as 变量3
            针对类型3,4,5的异常进行处理
        except:
            如果上述异常类型都没有匹配上，通过最后这个except代码块对异常进行处理

    说明：
        如果没有发生异常，程序会忽略except中的代码，继续向下执行。
        如果发生了异常，会忽略try中剩余代码，根据异常类型匹配到相应的 except 并执行其中的代码。
        如果发生了异常，且异常类型无法和任何except匹配，异常将向外传递。
       一个except可以同时处理多个异常，将这些异常放在一个元组中。
       最后一个 except 可以忽略异常类型，它将被作为通配符使用

4.else
    如果try块中的代码，没有发生异常，将会执行else块中的代码
    语法：
        try:
            可能发生异常的代码块
        except 类型 as e1:
            发生异常后，对异常处理的代码块
        except 类型 as e1:
            发生异常后，对异常处理的代码块
        except:
        ......
        else:
            try中没有发生异常，执行的代码块
    说明：
        从执行效果上说，将代码放到else块和直接放到try块中是一样的。
        将try正常执行完毕而没有引发任何异常后被执行的代码放到else中。提供了一种清晰的逻辑区分，将正常情况的代码与异常处理代码分开，使代码更易于理解和维护，有助于代码的可读性和可维护性。
            例如，你希望在try块中有些操作执行成功后，再执行其它代码，那就可以把代码放到else语句块中。

5.finally
    可选地，放在异常处理代码块最后
    表示不管是否发生异常，都要执行的代码
    一般用于资源的释放、连接的关闭等
    语法：
        try:
            可能发生异常的代码
        except 异常类型1 as 变量名1:
            异常处理的代码
        except 异常类型2 as 变量名2:
            异常处理的代码
        else:
            没有异常时执行的代码
        finally:
            无论是否发生异常都会执行的代码
    说明：
        -如果从执行效果上说，大部分场景，将代码放到finally语句和放到try-except块外效果是一样的。
        -finally 语句块是 try-except 结构的一部分，它确保了无论 try 块中是否发生异常，
        也无论 except 块是否被执行，其中的代码都会被执行。
        -直接放在try-except结构外面的代码只会在try-except结构正常执行完毕后才会执行，
        如果在try块中出现异常且没有被except块捕获，或者在except块中出现了新的异常导致程序终止，
        那么这部分代码将不会被执行。

6.raise
    当你想要在代码中明确表示发生了错误或异常情况时，可以使用 raise 来抛出异常。
    这可以帮助你在满足某些条件时停止程序的正常执行，并将控制权转移到异常处理部分。

7.assert
    assert用于判断一个表达式，在表达式条件为False的时候触发异常，常用于调试程序
    语法：
        assert 表达式 [,异常描述]
    等价于：
    if not 表达式:
        raise AssertionError([异常描述])

8.自定义异常
    语法：
        class 自定义异常类型(Exception):
            def __init__(self,value):
                self.value = value


9.异常的传递
    当存在 try 嵌套或函数嵌套时，若内层出现了异常且在内层无法处理，会将异常一层一层向外传递，
    直到异常被处理或程序报错 。
    传递的方向：沿着方法调用的方法或者程序执行的方向反向传递

10.with关键字
    封装了try...except...finally编码规范
    可以对资源进行自动的回收
    语法：
        with expression as variable:
            # 代码块
    说明：
        expression：通常是一个对象或函数调用，该对象需要是一个上下文管理器，即实现了__enter__和__exit__方法。
        variable：是可选的，用于存储expression的__enter__方法的返回值
    工作原理：
        使用 with 关键字系统会自动调用 f.close() 方法， with 的作用等效于 try finally 语句。
        当执行with语句时，会调用expression对象的__enter__方法。
        __enter__ 方法的返回值可以被存储在 variable 中（如果有），以供 with 代码块中使用。
        然后执行 with 语句内部的代码块。
        无论在代码块中是否发生异常，都会调用 expression 对象的__exit__ 方法，以确保资源的释放或清理工作，这类似于 try-except-finally 中的 finally 子句。
