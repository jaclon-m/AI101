
1.文件操作
    打开文件（建立和磁盘文件之间的通道）
        f = open("路径","模式","编码")
        路径：
            相对路径
            绝对路径
        模式：
            模式	                    说明
            r	        读写方式：只读，文件若不存在会报错。默认此模式
            w	        读写方式：写入，写入前清空原有数据。文件不存在会创建文件
            a	        读写方式：追加写入，在原有数据后追加，文件不存在会创建文件
            x	        读写方式：创建新文件并写入，文件若已存在会报错
            b	        编码方式：以二进制打开。一般用于非文本文件如图片等
            t	        编码方式：以文本模式打开，默认此模式
            +	        能读能写

    读取数据
        f.read()
        f.read(size)
        f.readLine()
        f.readLines()
    写数据
        f.write()
    关闭通道
        f.close()

6.__init__方法
    不需要程序员显示去调用
    在创建对象的时候自动调用__init__方法
    在定义类的时候，__init__不是必须的，如果没有显示的提供__init__方法，底层由默认的__init__
    因为__init__是在创建对象的时候自动调用的，所以也叫构造函数
    如果想在创建对象的时候，同时指定对象相关的实例属性，并对实例属性进行初始化，可以通过__init__进行传参，
    这个时候，要求创建对象的时候传递的参数个数和__init__中参数形参的个数必须要一致(self除外-底层在自动调用__init__方法的时候，
    会将当前对象放在第一个参数位置传递进来)
    所有的实例属性都定义在该方法中
    例如：
        def __init__(self,name,age):
            self.name = name
            self.age = age

        Person("zs",20)
    注意：
        self是一个约定俗成的名字，一般不需要我们去关心，可以通过self去调用当前类中其它实例属性或者方法
        __init__不能有返回值  只能是None

7.self参数
   -self作为实例传参
        在__init__方法以及其它的实例方法中，第一个参数都是self
        self代表类的实例自身。调用实例方法时，实例对象会作为第一个参数被传入。
        因此，我们调用p.eat()时就相当于调用了Person.eat(p)。
   -通过self在类中调用类的实例属性和实例方法

8.属性
    类属性
        也叫类变量，在类中方法外面进行定义
        类中所有的实例共享这个类属性
        访问方式：
            类名.属性名   或者     实例名.属性名
        通过 类名.属性名 添加与修改类属性
        若使用 实例名.属性名 则会创建或修改实例属性，和类属性没有关系。因此不建议类属性和实例属性同名。

    实例属性
        也叫实例变量。在类的__init__方法中定义的属性
        虽然定义在类中，每一个实例都有自己各自的属性值，相互之间不影响
            通过 self.属性名定义。
        访问方式：
            实例名.属性名
        通过 实例名.属性名 添加与修改实例属性
            只是针对当前实例添加，对其它的实例没有影响
        每个实例独有一份实例属性
9.方法
    实例方法
       是通过一类对象共有的行为抽取出来的
       实例方法在类中定义，第一个参数为self，代表实例本身。
       实例方法只能被实例对象调用。
       可以访问实例属性、类属性、类方法。
    类方法
        类方法在类中通过 @classmethod 定义，第一个参数为cls，代表类本身。
        类方法可以被类和实例对象调用。
        可以访问类属性。
        在不创建实例的情况下调用，通过类名直接调用，非常方便，适合一些和类整体相关的操作
    静态方法
        静态方法在类中通过 @staticmethod 定义
        不访问实例属性或类属性，只依赖于传入的参数
        可以通过类名或实例调用，但它不会访问类或实例的内部信息，
        一般用在工具类中，只是为了方便组织代码，把它放在了类里面。
